# Hivemind — Electron main process & context bridge

> Line numbers in this doc are approximate and drift as files change; symbol and
> channel **names** are the stable reference. Verify with grep before editing.

## Purpose

The main process (`main.js`) is the privileged half of the app. It owns:

- **Window lifecycle** — one `BrowserWindow` (`createWindow`, `main.js:291`), loading
  `src/index.html` over `file://`, with menu removed, navigation locked down, and
  mic-only permission handling.
- **PTY spawning** — one `node-pty` pseudo-terminal per pane, auto-typing a `claude`
  (or `codex`, or arbitrary) startup command with model/permission/resume/session
  flags composed from pane options (`spawnPty`, `main.js:155`).
- **IPC hub** — every renderer capability (git, files, plans, todos, prompt history,
  transcripts, usage, builds, notifications, autocorrect, speech models) is a channel
  registered here, mostly inside `app.whenReady()` (`main.js:575`).
- **`hm://` custom protocol** — serves speech-to-text assets (worker, transformers.js,
  ONNX models) to the `file://` renderer, which cannot `fetch()` `file://` URLs
  (`registerHmProtocol`, `main.js:76`).
- **Persistence** — `boards.json` in userData, temp image files, staged attachments.
- **Filesystem watching** — a single debounced recursive `fs.watch` on the active
  board's directory (`setWatch`, `main.js:263`).
- **Autocorrect service** — an nspell/Hunspell pass answering synchronous
  `spell:correct` queries (`main.js:428`–`573`).
- **Self-update** — `updater.checkForUpdates(mainWindow)` (`main.js:935`); portable
  builds offer to download the newest GitHub release. The updater talks to the user
  via native `dialog.showMessageBox`, not renderer IPC.

`preload.js` is the context bridge: it exposes `window.api` (all channels wrapped as
functions) to the sandboxed renderer. The renderer has `nodeIntegration: false`,
`contextIsolation: true` — it can only do what `window.api` offers.

## Key concepts

**Boards, panes, PTYs.** A *board* ("hive") is a project directory plus its saved
pane layout; the whole board list is a JSON array the renderer owns and persists
wholesale via `boards:save`. A *pane* is one terminal; each live pane maps to one
PTY entry in the `ptys` Map (`id -> { proc }`, `main.js:144`). Pane ids are
generated by the renderer and are the join key for `pty:*` channels.

**How Claude CLI processes start.** `spawnPty` (`main.js:155`) does NOT exec the
agent CLI directly. It spawns the default shell (`powershell.exe` on Windows via
`COMSPEC` sniffing, else `$SHELL`/bash — `defaultShell`, `main.js:146`) with
`useConpty: true`, then ~600 ms later *types* the startup command plus `\r` into the
PTY. The command is built by string surgery on the configured `startupCommand`
(default `claude`):

- `model` → inserts `--model <id>` after the `claude`/`codex` token (validated
  `[a-z0-9.-]+`).
- `permissionMode` → `--permission-mode acceptEdits|plan` or
  `--dangerously-skip-permissions` for "bypass"; only for `claude` commands.
- `resume` → `--resume <sessionId>` (validated id) or `--continue` (boolean).
- `sessionId` (fresh sessions only, strict UUID check) → `--session-id <uuid>` so
  the transcript file name is known up front. Never combined with resume.
- `initialPrompt` → appended as a shell-quoted positional argument (control chars
  stripped; PowerShell `''` vs POSIX `'\''` escaping). This is the only safe
  delivery — typing it later could hand it to the shell if the CLI is slow.

**Environment variables.** The PTY inherits `process.env` unchanged. The one env
var `main.js` itself honors is **`HM_USER_DATA`** (`main.js:12`): if set *before*
launch, `app.setPath('userData', ...)` redirects the entire userData tree. This is
the sanctioned way to run a test instance without touching the user's real
`boards.json`/models — the user's live instance is usually running, so always
isolate dev runs with it.

**userData layout** (default `%APPDATA%\hivemind`, or `HM_USER_DATA`):

- `boards.json` — the board array (see Persistence).
- `models/<repo>/…` — speech models downloaded on demand by `stt:ensureModel`;
  shadows the bundled `models/` dir via the `hm://models` root ordering.

**`hm://` protocol.** Registered as privileged before ready (`main.js:71`), handled
in `registerHmProtocol`. Hosts map to ordered on-disk root lists (`HM_ROOTS`,
`main.js:41`): `hm://app` → `src/`, `hm://vendor` → transformers.js dist,
`hm://models` → userData/models then bundled `models/`. Each candidate path is
traversal-guarded against its root; responses get explicit MIME types (`HM_MIME`)
and `Access-Control-Allow-Origin: *`. `STT_DOWNLOADS` (`main.js:52`) lists exactly
which files each downloadable model repo needs and must stay in sync with the
`STT_MODELS` registry in `src/renderer.js`.

**Renderer bootstrap values.** The sandboxed preload cannot `require('os')` or read
`package.json`, so main passes the Windows OS build number and app version through
`webPreferences.additionalArguments` (`--hm-os-build=`, `--hm-app-version=`,
`main.js:305`); preload parses them from `process.argv` (`preload.js:9`–`19`) and
exposes them as `api.osBuild` / `api.appVersion`.

## Persistence

Main-process reads/writes (helper modules add more inside each project's
`.hivemind/` and `~/.claude/`):

- **`<userData>/boards.json`** (`boardsFile`, `main.js:118`) — a JSON array of board
  objects, treated as opaque by main (the renderer defines the shape: name, dir,
  panes, layout, settings). Written pretty-printed on every `boards:save`.
- **`<userData>/models/<repo>/…`** — downloaded speech-model files
  (`stt:ensureModel`, `main.js:858`); served via `hm://models`.
- **`os.tmpdir()/hivemind-images/`** — pasted/dropped screenshots (`paste-*` /
  `clip-*` files); swept of >1-week-old files at startup (`sweepOldTempImages`,
  `main.js:945`).
- **`<project>/.hivemind/attachments/`** — staged attachment copies (`attach:stage`,
  `main.js:683`); also swept of >1-week-old files per call.
- Via helpers: `.hivemind/plans/*.md` + comment sidecars, `.hivemind/todos.json`,
  `.hivemind/prompt-history.json`, the project's `.gitignore` (ensureIgnored), and
  read-only tailing of `~/.claude/projects/**/*.jsonl` transcripts.
- **Chromium spell-check dictionary** — custom words added via the context menu are
  mirrored into nspell on load (`loadSpell`, `main.js:463`).

## Invariants & gotchas

- **`HM_USER_DATA` must be handled before anything touches `app.getPath('userData')`** —
  it's line 12 for a reason. Never launch a test instance without it: the user's
  live instance is usually running, shares the real `boards.json`, and must never
  be killed.
- **`spell:correct` must always set `event.returnValue`** — it's a synchronous IPC;
  a code path that returns without setting it hangs the renderer (`main.js:572`).
- **Protocol scheme registration order**: `protocol.registerSchemesAsPrivileged` and
  the `SharedArrayBuffer` feature switch must run before app `ready`; the `hm`
  handler itself installs in `whenReady`. Losing the SAB switch silently drops the
  voice worker to single-threaded WASM (~4x slower) with no error.
- **PTY startup is typed, not exec'd**: the 600 ms `setTimeout` before writing the
  command (`main.js:237`) exists so the shell prompt is ready. Anything that needs
  to reach the agent CLI at startup must ride the composed command string (like
  `initialPrompt` does) — writing extra input into the PTY later races the shell.
- **Flag-injection regexes are guards, not decoration**: model ids, session ids and
  resume ids are validated before splicing into the shell command; keep validating
  anything new that gets interpolated, since it is executed by a shell.
- **One fs watcher total**, keyed by directory (`setWatch` is a no-op for the same
  cwd). Failure to establish a recursive watch degrades silently to manual refresh.
- **`STT_DOWNLOADS` (main.js) and `STT_MODELS` (renderer.js) must stay in sync** —
  a model listed only in the renderer can never download.
- **Single-flight build**: `buildRunning` (`main.js:289`) guards `build:portable`; a
  failed build restores the previous version so failures don't burn version numbers.
  Always bump `package.json` version before a portable build — the artifact name
  embeds it and the updater compares it against releases.
- **Window teardown kills all PTYs** and disposes transcript tails on
  `window-all-closed` (`main.js:958`); there is no PTY persistence across restarts —
  panes are *re-created* from `boards.json` (optionally resuming Claude sessions
  via `--resume`/`--continue`).
- **Navigation lockdown** (`main.js:322`–`337`): the renderer must never navigate or
  open child windows (they'd inherit the preload). External links must go through
  `open:external` / the window-open handler, both restricted to http/https/mailto.
- **Dev vs packaged paths**: `__dirname` works in both because `asarUnpack` extracts
  `models/**` and the transformers.js dist; if you add a new on-disk asset that
  must be `fs`-readable or served over `hm://`, add it to `asarUnpack` in
  `package.json` or it will be unreadable inside the asar archive.
- **The transcript binder is heuristic** (transcript.js pairs panes to session
  JSONLs); it runs in the main process, so changes there need a full app relaunch,
  and a mis-bind shows the wrong conversation but never delivers prompts wrongly.
- **Sandboxed preload**: it cannot `require` Node built-ins. New values computed in
  main must travel via `additionalArguments` (like `--hm-os-build`) or IPC.

## How to extend

To add a new IPC feature (say, `foo:doThing`):

1. **main.js** — register the channel inside `app.whenReady()` next to its family
   (`ipcMain.handle('foo:doThing', (_e, { cwd, arg }) => foo.doThing(cwd, arg))`).
   Put real logic in a helper module (like `git.js`/`plan.js`), not inline.
   Validate anything that touches a shell command or a filesystem path.
2. **preload.js** — add a wrapper in the `contextBridge.exposeInMainWorld('api', …)`
   object: `foo: { doThing: (cwd, arg) => ipcRenderer.invoke('foo:doThing', { cwd, arg }) }`.
3. **renderer** — call `window.api.foo.doThing(cwd, arg)` from `src/renderer.js`.
4. For main→renderer streams, `mainWindow.webContents.send('foo:event', payload)`
   guarded by `mainWindow && !mainWindow.isDestroyed()`, plus an `onFooEvent(cb)`
   subscription in preload that returns an unsubscribe function (copy the
   `onBuildProgress` pattern, `preload.js:176`).
5. If the feature is user-facing, **update the Help modal** in `src/index.html`
   (`#help-modal`) in the same change — project rule from `CLAUDE.md`.
6. Main-process changes need a full app relaunch (no hot reload). Test with an
   isolated instance: set `HM_USER_DATA` to a scratch dir before launching.
