          ❔ Help
          
        
          
        
        
        
        
          
        
            
        Getting started
        
            
          
              
        1. Click ＋ next to the logo to create a hive and point it at a project directory.
          2. 
              
        2. Press ＋ Thread to open a Claude thread — open as many as you like; they all share the hive's directory.
          3. 
              
        3. Use the 🗂 Explorer and ⑂ Source Control buttons to browse files and manage Git.
          4. 
            
        
          
        

          
        
            
        Threads & broadcast
        
            
          
              
        • 📡 Broadcast — send the same command (or mirror your typing) to every thread on the hive.
          • 
              
        • 🎤 Voice typing — dictate into the focused thread; speech is transcribed on-device by Whisper (English, fully offline) and typed at the cursor when you pause. Toggle with the ~ key. Use the user dictionary in ⚙ Settings → Voice to fix words it mishears.
          • 
              
        • Each thread's header shows a caption built from the last prompt you send it, so you can see what every thread is working on at a glance. Double-click the name to rename a thread.
          • 
              
        • Change a thread's model or font from its header, or with A−/A+ and Ctrl+scroll.
          • 
            
        
          
        

          
        
            
        Source Control (Git)
        
            
          
              
        • Push ↑ — stages every change, commits (auto-drafting a message if you left the box empty), then pushes your work up to GitHub. The first time, it connects the repo to GitHub for you. (Ctrl+Enter in the commit message box does the same thing.) If the push is rejected because GitHub has newer commits, Pull first.
          • 
              
        • Pull ↓ — brings down commits made on GitHub (or another machine) and merges them into your branch.
          • 
              
        • ⋯ menu → Revert to GitHub — tucked into the ⋯ overflow menu next to Pull/Push. Discards all local changes and resets the branch to match GitHub. Cannot be undone.
          • 
            
        
          
        

          
        
            
        Shortcuts
        
            
          
              
        • ~ — start / stop voice typing
          • 
              
        • Ctrl+F — find in the focused thread
          • 
              
        • A− / A+ or Ctrl+scroll — change the focused thread's font size
          • 
              
        • Esc — close the open dialog
          • 
            
        
          
        

          
        
            
        Building Hivemind
        
            
        📦 Build Portable appears in the toolbar only when the active
              hive is pointed at the Hivemind source checkout. It runs electron-builder's portable target to
              produce a single self-contained .exe, then opens the dist folder when
              it finishes. While it runs, the button shows the current build stage and a live elapsed timer
              (a portable build typically takes a few minutes); hover it to see the latest build output line.
