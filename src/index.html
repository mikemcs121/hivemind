      About
      
              
      Hivemind —
      
            
      
          
      

          
          
      
            
      
              Dictate straight into the focused thread — transcribed text is typed at
              the cursor when you pause. Speech is recognized on-device (English),
              fully offline. Press the ~ key any time to start or stop
              listening. You can start speaking immediately — words spoken while the
              model is still loading are kept, and toggling off still types whatever
              you had just said.
            
      

            

            
      
              
              
              
            
      

            
      
              User dictionary
              Fix words speech keeps mishearing. Each phrase you say on the
                left is replaced with the text on the right before it is typed.

        Getting started
        
            
          
              
        1. Click ＋ next to the logo to create a hive and point it at a project directory.
          2. 
              
        2. Press ＋ Thread to open a Claude thread — open as many as you like; they all share the hive's directory.
          3. 
              
        3. Use the
                 Explorer
                and
                 Source Control
                buttons to browse files and manage Git, and the
                 Plan
                button to open the focused thread's plan review, and the
                 Todo
                button to keep a checklist for the hive, and the
                 Prompt History
                button to see the prompts you've sent in this hive — click one to jump to that prompt in the conversation where it appears (the thread is focused and the message flashes briefly). Hover an entry for its ↩ button to send it again to the focused thread, or 🗑 to remove it from the list.
          4. 
              
        4. Drag the sidebar's right edge to make it (and the Explorer, Source Control, Todo and Prompt History panels docked in it) wider or narrower. Double-click the edge to reset the width; it's remembered across restarts.
          5. 
            
        
          
        

          
        
            
        Threads
        
            
          
              
        • 🎤 Voice typing — dictate into the focused thread; speech is transcribed on-device (English, fully offline) and typed at the cursor when you pause. Toggle with the ~ key. You can start speaking right away — words spoken while the model loads are kept, and toggling off still types what you had just said. If the HUD says “Didn’t catch that”, the model heard you but couldn’t make out words. In ⚙ Settings → Voice you can pick the recognition model (the default Moonshine model is bundled; others download once, then run offline) and use the user dictionary to fix words it mishears.
          • 
              
        • Each thread's header shows a caption built from the last prompt you send it, so you can see what every thread is working on at a glance. Short replies like yes, no or a menu number don't replace the caption, so answering a prompt keeps the real task on show. Once a thread has a caption it takes over as the header title and the default claude # name is hidden. Double-click the title to rename a thread — a name you set yourself always stays shown.
          • 
              
        • Each thread can run a different agent — pick Claude, ChatGPT, or Gemini from the dropdown in the thread header. One hive can mix them (e.g. three Claude threads and one ChatGPT thread). Switching a running thread restarts it with the new agent. ChatGPT needs OpenAI's Codex CLI (npm install -g @openai/codex) and Gemini needs Google's Gemini CLI (npm install -g @google/gemini-cli) installed and signed in.
          • 
              
        • Claude and ChatGPT threads each have a model dropdown in the header. On a Claude thread, switching a running thread changes the model live (via /model); on a ChatGPT thread the model is a Codex startup flag, so switching a running thread restarts it with a fresh conversation. The last choice is remembered as the default for new threads of that agent (also settable in ⚙ Settings → General). Gemini threads have no model dropdown.
          • 
              
        • Each Claude thread has a permission mode dropdown in its header — Default, Accept Edits, Plan, or Bypass ⚠ — so you can see and set the mode without hunting for it in Claude's own display. It's handed to Claude Code as a startup flag (Bypass skips the accept-permissions screen entirely). Since Claude has no command to change mode mid-session, switching a running thread restarts it (resuming the conversation when there is one). Bypass turns the dropdown amber as a reminder that permissions are being skipped. The choice is remembered as the default for new threads.
          • 
              
        • When a thread finishes its turn, its whole pane pulses green and the header status shows ✓ done until you click into that thread — so completed work stands out across a busy hive. The hive list's status dot and the maximize tab strip's dots pulse green too. (Threads that need an answer or hit an error still pulse amber/red as before.)
          • 
              
        • Each Claude thread's header shows a ~$ cost estimate: what the conversation so far would have cost at Claude API list prices if it weren't covered by your plan (it's informational — nothing is billed). Hover it for a per-model breakdown of tokens and cost. It counts the whole conversation (including what happened before a restart or resume), resets when /clear starts a new session, and doesn't appear on ChatGPT or Gemini threads.
          • 
              
        • Change a thread's model or font from its header, or with A−/A+ and Ctrl+scroll. A narrow thread hides its lower-priority header controls (font buttons first, then the permission, model, and agent dropdowns, then the view toggle) so the ⛶ maximize and ✕ close buttons always stay reachable — widen or maximize the thread to get the hidden controls back.
          • 
              
        • ⛶ Maximize (or Ctrl+Enter) blows the focused thread up to fill the hive. A tab strip appears across the top listing every other thread — each with a live status dot — so you can see what's hidden behind it and click a tab to jump straight to that thread. Press Ctrl+Enter or ⛶ again to restore the tiled layout.
          • 
            
        
          
        

          
        
            
        Chat view
        
            
          
              
        • Claude and ChatGPT threads open in a chat view: a clean transcript of the conversation — your messages, the agent's replies as formatted markdown, and collapsible tool calls — instead of the raw terminal. The real terminal keeps running underneath, untouched; click >_ in the thread header to reveal it (find and scrollback live there) and 💬 to come back. The choice is remembered per thread. Gemini threads are terminal-only.
          • 
              
        • The bar at the top shows the conversation topic — on Claude threads it's Claude Code's own running summary of the session (once it has one); on ChatGPT threads it's the first message you sent the conversation.
          • 
              
        • On Claude threads, click 🕘 History in the top bar to browse this thread's past conversations. The menu lists earlier sessions in the project (newest first, titled by their topic or first message); pick one to view it over the live chat. To continue a past conversation, just type in the composer and send — the thread restarts on that conversation (claude --resume) with your message as the next turn. Click Back to live (or the live entry in the menu) to return to the running conversation instead, which keeps updating underneath the whole time.
          • 
              
        • While the thread is working on a turn, a spinning 🌀 swirl appears below the last message; it disappears when the turn finishes.
          • 
              
        • Hover any of the agent's replies to reveal a small ⧉ Copy button at the end of the message — click it to copy that message's raw markdown to the clipboard (it flips to ✓ briefly to confirm). Each code block gets its own ⧉ button in its top-right corner (revealed on hover) that copies just that block's code.
          • 
              
        • When Claude asks you a multiple-choice question, it appears as a question card in the chat — click an option to answer (it presses that option's number in the hidden terminal; multi-select options show a ☑ as you toggle them — click Review ⇥, and Claude Code's confirmation step appears as its own card where Submit answers sends your picks). The thread shows needs you until the question is answered. For a custom answer, click Open terminal on the card and type it there. Claude Code's plan-approval menu (“Ready to code?” / “Would you like to proceed?”) appears as the same kind of card — with the plan itself rendered right on the card (a collapsible 📋 section above the options, so you can read what you're approving without opening the terminal) — click an option to approve the plan or keep planning, or click the card's 📋 Review plan button to open the full plan review window (comments and request-changes live there). ChatGPT approval prompts (run a command, make edits, grant permissions, network access) appear as the same kind of card too, showing what's being approved with clickable options.
          • 
              
        • Filter chips at the top right (Tools, Thinking, Meta, Subagents) control exactly what the chat shows. Tool calls appear as collapsed one-liners — click one to expand its input and result. Your choices persist per thread and become the default for new threads.
          • 
              
        • The composer at the bottom sends to the thread: Enter sends, Shift+Enter adds a line — multi-line messages arrive as one message. Press ↑ in an empty composer (or on the first line) to recall messages you've sent before, ↓ to come back — editing keeps whatever you land on. Misspelled words get a red squiggle — right-click for suggestions or to add a word to the dictionary — and clear typos are autocorrected the moment you finish a word (Ctrl+Z undoes a fix; Add to dictionary protects a word; turn it off in ⚙ Settings → General → Typing).
          • 
              
        • Click ⏹ Interrupt next to the composer's Send button to stop whatever the thread is running — it sends the same Ctrl+C the terminal would, without you having to open the terminal view first. Handy for cutting off a long-running command or a turn that's gone down the wrong path.
          • 
              
        • Attachments: paste an image (including Win+Shift+S screenshots), drag files onto the chat, or click 📎 to pick files — each shows as a chip above the input (images get a thumbnail preview). Click a chip's ✕ to remove it; when you send, the attached files' paths go along with your message so the agent can read them. On ChatGPT threads, images are handed to Codex as real image input (they appear as [Image #N] in its composer, so the model can actually see them), and other files from outside the project are first copied into .hivemind/attachments/ in the project (the Codex sandbox can only read files inside its workspace); the copies are git-ignored and swept after a week.
          • 
              
        • The composer autocompletes as you type: start a message with / for slash-command suggestions (Claude Code built-ins plus your project's .claude/commands and .claude/skills), or type @ anywhere for file paths from the project directory — completing a folder steps into it. ↑/↓ pick a suggestion, Tab or Enter accepts it, Esc dismisses the menu.
          • 
              
        • When the hidden terminal needs something the chat can't show — a permission prompt, an approval menu, an error, or a turn that ended on a question — it appears as a card in the chat flow (right where the conversation is), with quick keys (1, 2, Enter, Esc) and an Open terminal button showing the real prompt. While a prompt is waiting the composer is locked — a typed message wouldn't reach the terminal (it wants a menu keypress), so answer with the card's keys or open the terminal for a custom reply; the composer unlocks the moment the prompt is answered. This works on both Claude and ChatGPT threads (ChatGPT approvals appear while the thread still looks busy, and recognized ones render as full question cards instead).
          • 
              
        • If the agent hits an API error — the chosen model isn't available on your account, a usage limit was hit, the request failed — the full error message appears in the chat as a red error card (it's never hidden by the filter chips), and the thread's status flips to error so you're notified.
          • 
              
        • The chat is read from the thread's local session log — Claude Code's transcript for Claude threads, the Codex CLI's session file for ChatGPT threads. If Hivemind can't find a Claude transcript (e.g. a custom startup command that isn't claude), a notice appears; a ChatGPT chat simply fills in from your first message (Codex writes its log when the conversation starts). The terminal view still works as always — and if you click the notice's Open terminal to watch a thread while its transcript is still catching up, the chat view snaps back automatically the moment the transcript is found (opening the terminal that way is temporary, so the thread won't reopen stuck in the terminal later; the >_ / 💬 toggle is still the way to make a lasting choice).
          • 
            
        
          
        

          
        
            
        Hivemind commands
        
            
        Start a composer message (or a single dictated sentence) with
              “Hivemind” and the rest is a command Hivemind itself carries out instead of
              text sent to Claude — e.g. “Hivemind, open a new thread and fix the failing login test”
              or “Hivemind, increase the font size”. A toast at the bottom of the workspace confirms
              what it did. Anything that starts with “Hivemind” is treated as a command — if it isn't
              one Hivemind recognizes, the toast says so and nothing is sent to the thread. Mentioning
              Hivemind mid-sentence is sent to the thread as normal.
        
            
        
            
        “This thread” means the one whose composer you typed into (or the focused
              one when dictating). If voice typing mishears the wake word (e.g. “hive mine”), add a
              correction in ⚙ Settings → Voice's dictionary.
        
          
        

          
        
            
        Source Control (Git)
        
            
          
              
        • Push ↑ — stages every change, commits (auto-drafting a message if you left the box empty), then pushes your work up to GitHub. The first time, it connects the repo to GitHub for you. (Ctrl+Enter in the commit message box does the same thing.) If the push is rejected because GitHub has newer commits, Pull first.
          • 
              
        • Pull ↓ — brings down commits made on GitHub (or another machine) and merges them into your branch.
          • 
              
        • ⋯ menu → Revert to GitHub — tucked into the ⋯ overflow menu next to Pull/Push. Discards all local changes and resets the branch to match GitHub. Cannot be undone.
          • 
              
        • Once the repo is connected to a remote, the GitHub icon next to the branch name opens the project's page on GitHub in your browser.
          • 
              
        • The ↓/↑ counters next to the branch name show commits to pull down from / push up to GitHub. Hivemind checks GitHub for new commits in the background while the panel is open (every few minutes); hit ⟳ to check right now.
          • 
            
        
          
        

          
        
            
        Plan review
        
            
          
              
        • When a Claude thread works in plan mode (the Plan permission mode, or whenever Claude plans on its own), Hivemind detects the plan automatically. While Claude drafts it the thread's header shows a 📋 planning chip; the moment Claude finishes and asks for approval (“Ready to code?” / “Would you like to proceed?”), the plan pops up as a full document — rendered markdown in an editor-style window that live-updates as the plan is written. If the thread isn't focused, nothing pops up: the header shows a 📋 plan ready chip instead — click it (or the sidebar's Plan button) to open the review. The auto-popup can be turned off in ⚙ Settings → General → Plan review.
          • 
              
        • Approve · auto-accept and Approve · manual edits answer Claude Code's own approval menu in the hidden terminal — exactly like picking “Yes, auto-accept edits” (or its equivalent in the thread's permission mode, e.g. “Yes, and bypass permissions” — hover the button to see which) or “Yes, manually approve edits” there. The status chip flips to Approved once Claude confirms, and the thread starts implementing. The same approval menu also shows up as a clickable card in the thread's chat view — with the plan rendered on the card and a 📋 Review plan button that opens this review window — so you can read the plan and answer right there.
          • 
              
        • Inline feedback: highlight any text in the plan and click ＋ Comment to attach a note to it — like review comments on a document. Comments are saved with the project (in .hivemind/plans/, kept out of Git), so they survive restarts.
          • 
              
        • Request changes picks “No, keep planning” in the thread and sends your comments (each with the text it refers to, plus the optional overall-feedback line) as the revision request. Claude revises, the document updates live, and comments you already sent show collapsed with a sent tag. Comments whose text no longer appears in the revised plan are marked (not found in current plan).
          • 
              
        • Plans render as rich markdown — headings, tables, links (which open in your browser), block quotes and task lists. On plans requested via ⟳ (see below) you can tick a checkbox in a - [ ] task list to mark it done — the change is written back to the plan file. Native plan-mode files belong to Claude Code, so their checkboxes are read-only.
          • 
              
        • ChatGPT and other threads (no native plan mode): click ⟳ in the review window to ask the thread to write its plan to .hivemind/plans/ in your project — it renders in the same document view with the same commenting, and Request changes types the comments straight into the thread.
          • 
              
        • ⧉ copies the plan's raw markdown to the clipboard. Esc, ✕, or clicking the backdrop closes the review; the plan stays available on its thread.
          • 
            
        
          
        

          
        
            
        Todo
        
            
          
              
        • The Todo panel is a simple checklist for the hive. Unlike the Plan panel it's shared across every thread in the hive, not tied to the focused one.
          • 
              
        • Type in the box at the top and press Enter (or ＋) to add an item. Tick the checkbox to mark it done, double-click an item to rename it, and hover to reveal a ✕ to delete it. Spelling is checked as you type — typos get the same red squiggles, right-click suggestions, and autocorrect as the chat composer.
          • 
              
        • Push a todo to a new thread: hover an item and click the ▶ to open a fresh thread that starts working on it right away — the item's text (plus any unfinished sub-items) becomes the new Claude's first prompt. The todo stays unticked until you mark it done yourself.
          • 
              
        • Quick add from anywhere: start a composer message (or a dictated sentence) with todo — e.g. “todo make the repo public” — and it's added to the panel instead of being sent to Claude (TODO: and to-do work too; a bare todo just opens the panel). “Hivemind, add a todo …” does the same. If dictation hears it as two words (“to do”), add a Settings → Voice dictionary correction mapping it to “todo”.
          • 
              
        • Items nest to any depth: hover a row and click the ＋ to add a sub-item beneath it. Ticking a parent ticks all of its sub-items, but sub-items never auto-tick their parent — a parent is only done when you tick it yourself (un-ticking a sub-item does un-tick a done parent); a partly-done parent shows a dash. Use the ▾ / ▸ caret to collapse or expand a branch.
          • 
              
        • The footer shows how many leaf items are done and offers Clear completed to remove the ticked ones (a done parent stays if it still has unfinished sub-items).
          • 
              
        • The list is saved with the project in .hivemind/todos.json (kept out of Git via .gitignore, like plans), so it survives restarts and updates live if a thread edits the file.
          •

          •

          •

          •

        Usage & limits
        
            
        The Usage pill in the toolbar shows how much of your
              Claude plan's most-constrained limit is used right now (green → yellow → red as you approach
              it). Click it for the full picture: the 5-hour session window and weekly limits with their
              reset times — the same numbers as Claude Code's /usage — plus how many tokens
              each model has used today, totalled from your local Claude Code transcripts. Figures refresh
              about once a minute.
        
          
        

          
        
            
        Building Hivemind
        
            
        The 📦 Build portable copy button in
              Settings → General → Building Hivemind appears only when the active
              hive is pointed at the Hivemind source checkout. It bumps the patch version in
              package.json, runs electron-builder's portable target to produce a single
              self-contained .exe, then publishes it as a GitHub release (tag
              vX.Y.Z, via the gh CLI) and commits & pushes the version bump —
              this is what portable copies in the wild auto-update from. The dist folder opens
              when it finishes. While it runs, the button shows the current stage and a live elapsed timer
              (a build typically takes a few minutes); hover it to see the latest output line. If publishing
              fails the build itself is kept; a failed build restores the old version number.
        
          
        

          
        
            
        Updates (portable build)
        
            
        When you run the portable .exe, Hivemind checks GitHub for a
              newer release on startup. If one exists it asks once; click Update Now and it
              downloads the new version next to the current file, restarts into it, and automatically
              removes the old .exe along with any other older versions sitting in the same
              folder. Boards and settings are preserved. Installed and source
              (npm start) runs never check.
        
          
        

          
        
            
        Appearance
        
            
        Pick a colour Theme in ⚙ Settings → General
              (Midnight, Forest, Ember, Grape, plus light themes Paper & Rose). It recolours the whole
              app — including every terminal — instantly, and is remembered across restarts.
        
          
        

          
        
            
        More
        
            
        Open ⚙ Settings for the theme, defaults, notifications, and voice options.
              The General tab's About section shows the Hivemind version you're running.
